# یادداشت معماری و Migration نسخه 5

- مشتری هنوز داده واقعی وارد نکرده است؛ بنابراین ریسک migration عملی پایین است.
- دیتابیس هنگام اولین اجرا جدول‌ها و ستون‌های جدید را با `CREATE TABLE IF NOT EXISTS` و `ALTER TABLE` اضافه می‌کند.
- شناسه Tauri همچنان `com.rezo.persian-dietitian` است تا مسیر داده و تنظیمات نسخه قبلی حفظ شود.
- مسیر فایل داخلی `DietoyData` عمداً تغییر نکرده تا فایل‌های نسخه‌های قبلی گم نشوند؛ نام نمایشی برنامه Dietory است.
- ویزیت‌های قدیمی بدون `track_id` در UI با منطق سازگاری نمایش داده می‌شوند و ویزیت‌های جدید به مسیر واقعی متصل می‌شوند.
